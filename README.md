# astrbot_plugin_toyoko_watch

在 AstrBot 内监控东横 INN 空房，并通过 OneBot v11 主动私聊 QQ、发送到 QQ 群，或使用 SMTP 邮件通知。插件自带 WebUI，可检索全酒店、选择日期与房型，并把“单人房”和“多人房”等需求分别管理。

本插件不依赖 `win11toast`，也不会抑制第一次查到的空房。第一次成功检查就有符合条件的房间时，会立刻通知。

## 功能

- 本地保存 352 家东横 INN 酒店的编号、名称、地区和地址；支持按编号、名称、城市或地址检索。
- 初次安装提供两条暂停的横滨任务，默认选择 `00075` 横滨体育场前 1 和 `00073` 横滨体育场前 2。
- 一个任务可同时选择多家酒店。
- 支持入住和退房日期、1 至 4 人、禁烟/吸烟、一般/会员库存。
- 房型分为单人、多人两大类，并支持经济单人、标准单人、单人大床、经济双人、双人大床、双床和三人房。
- 可从官网探测实际日文房型名，也可填写精确名称和自定义日文关键词。
- 一个任务包含多个独立需求。实际订到后手动把对应需求标记为“已订到”，其他需求继续运行。
- QQ 私聊、QQ 群聊和 SMTP 邮件可组合使用，并提供测试按钮。
- 首次命中立即通知；持续不变的库存保持安静；无房后重新出现会再次通知；请求错误不会误判为无房。
- 每个任务可设置 60 至 3600 秒的检查间隔，官网请求带并发限制和随机抖动。

## 要求与安装

- AstrBot `4.26.0-beta.8` 或更高版本。
- OneBot v11 平台使用 AstrBot 的 `aiocqhttp` 适配器。
- Python 3.10 或更高版本。

在 AstrBot 插件管理页从仓库安装：

```text
https://github.com/huvz04/astrbot_plugin_toyoko_watch
```

也可以把仓库克隆到 AstrBot 的 `data/plugins/astrbot_plugin_toyoko_watch`，然后重载插件。AstrBot 会根据 `requirements.txt` 安装 `aiohttp`；若当前部署关闭了自动安装，请在 AstrBot 的 Python 环境中手动执行：

```bash
pip install -r data/plugins/astrbot_plugin_toyoko_watch/requirements.txt
```

## 第一次配置

1. 确认 OneBot v11 适配器已连接。
2. 在希望接收通知的 QQ 私聊或群聊中发送 `/toyoko bind`。该命令会自动创建正确的通知目标。
3. 打开插件详情中的“空房监控”页面。
4. 编辑一条初始任务，或新建任务；选择酒店、日期、独立房间需求和通知目标。
5. 点击目标旁的“测试”，确认主动消息能够收到。
6. 点击“立即检查”做一次真实官网检查，确认无错误后再启用任务。

也可以在 WebUI 手动添加目标：

- 私聊：QQ 号会生成 `aiocqhttp:FriendMessage:<QQ号>`。
- 群聊：群号会生成 `aiocqhttp:GroupMessage:<群号>`。

## 房间需求

每条需求都是独立的。例如，同一个任务可以同时包含：

- “我的单人房”：1 人，单人大类，选择经济单人、标准单人和单人大床。
- “两人房”：2 人，多人大类，选择经济双人、双人大床和双床。

“探测官网房型”会读取所选酒店和日期下官网列出的实际房型名，即使某房型当前库存为零也会显示。可以把日文名称填入“官网精确房型名”，减少启发式分类造成的偏差。

插件只报告库存，不会自动订房。实际预订成功后，在任务列表把相应需求改成“已订到”。恢复为“监控中”后，下一次有房会被视作新的首次命中并立即通知。

## 邮件

SMTP 密码只保存在 AstrBot 插件配置中，不会写入仓库、状态接口或日志。可配置：

- SMTP 主机和端口。
- 隐式 SSL，或连接后的 STARTTLS。
- 用户名、密码/应用专用密码和发件人。
- 一个或多个收件人。

开启全局 `smtp_enabled` 后，还要在具体任务中勾选“同时发送邮件”。先在 WebUI 点击“发送测试邮件”验证配置。

## 指令

- `/toyoko status`：查看调度器、任务和最近检查状态。
- `/toyoko bind`：把当前私聊或群聊添加为通知目标（管理员）。
- `/toyoko test`：向当前会话发送主动测试消息（管理员）。
- `/toyoko check`：立即检查全部已启用任务（管理员）。
- `/toyoko catalog-refresh`：从官网刷新并验证酒店目录（管理员）。
- `/toyoko help`：查看简要帮助。

## 数据与故障处理

可变数据保存在 AstrBot 插件数据目录下的 `astrbot_plugin_toyoko_watch` 子目录：

- `hotels.json`：最后一次验证通过的酒店目录。
- `tasks.json`：监控任务与需求状态。
- `targets.json`：QQ 通知目标。
- `state.json`：最近检查、空房状态和通知重试记录。

目录刷新会先验证数量、编号和默认横滨酒店，再进行原子替换。刷新失败时继续使用旧目录。JSON 文件损坏时会先备份损坏副本，再恢复安全默认值。

若“有房但没有通知”，依次检查：

1. `/toyoko status` 中任务是否启用、是否有活动需求。
2. 任务是否勾选了已经启用且测试成功的 QQ 目标，或完整的 SMTP 配置。
3. WebUI 的最近错误中是否出现官网超时、页面结构变化或 OneBot 离线。
4. 房型人数、吸烟偏好和一般/会员库存筛选是否过窄；可先用“探测官网房型”核对日文名称。
